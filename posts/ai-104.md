# Entropy Loss
Deep Learning에서 흔히 사용되는 Entropy

Binary Cross Entropy (BCE) : 아주 많은 것 중 하나의 케이스
Cross Entropy (CE) : BCE를 generalization 한 것. BCE는 CE의 special case
-> output node 숫자에 따라 결정 한다고 해도 크게 어긋나지 않음

Binary Classification의 경우 일반적으로 예측 결과 (Prediction)과 정답 (Label)의 차이를 줄이는 방식으로 학습을 한다.

![이미지 설명](/assets/binary_classification.jpg)

$L(\theta)=\sum_{i=1}^n(\hat{y}_i-y_i)^2$

이 보다 더 좋은 방법으로는 Entropy를 사용 할 수도 있다. 예측 전략을 바꾸는 것인데 sigmoid를 통과한 값이 0.이상이면 True, 0.5 미만이면 False로 예측하는 모델을 구축하고 loss에 cross entropy를 사용하는 것이다.

Label 값이 True였다고 가정해보자. Sigmoid를 통과한 값이 $\hat{y}=0.6$이라고 가정해보면, 0.5 이상이므로 True라고 판단할 것이다. 그런데 생각해보면 이게 진짜 잘 맞춘건가? 0.5000001도 True라고 예측했다면 맞긴한데 100 % 잘 맞췄다고 보기엔 부족함이 있다.
Entropy 관점에서 True가 정답인 상황은 확률 분포가 2개이다.

Entropy: $H(X)=E(I(X))=-\sum_{i}p_i\times logp_i$

![이미지 설명](/.jpg)

정답 확률분포는 $E(Y)=-(1\times log1)-(0\times log0)=0$이고 예측값 확률분포는 $E(\hat{Y})=-(0.6\times log0.6)-(0.4\times log0.4)=0.971$이다. 즉, Entropy 관점에서 보면 얼마나 틀렸는지, 틀릴수 있는 기대값을 표현할 수 있다. False가 정답인 상황은 True인 상황의 반대이므로 계산해보면 동일하게 0, 0.971이 나온다. 따라서 Cross Entrpoy 개념이 필요하다.

Cross Entropy: $H(X,Y)=E_X(I(Y))=-\sum_{i\in X}P(X=i)\times logP(Y=i)$ (확률 X Random Variable)
Binary classification의 경우 Cross Entropy는 $H(Y,\hat{Y})=-P(Y=True)\times (logP(Y\hat{Y}=True)-(P(Y=False))\times log(P(\hat{Y}=False))$

Binary Classification 문제에서 출력을 한번에 처리 하려면,
$H(Y,\hat{Y})=-P(Y=True)\times (logP(Y\hat{Y}=True)-(P(Y=False))\times log(P(\hat{Y}=False))$ 이므로,
Sigmoid 출력이 성공확률 p이기 때문에 최종 Binary Cross Entropy 수식은
$BCE=-[plogp+(1-p)log(1-p)]$
![BCE](/assets/BCE_combined.jpg)
정답에서 가까워질수록, 멀어질수록 entropy는 어떻게 변하는지 관찰해보면 loss function에 적용할 수 있음을 알 수 있다.
