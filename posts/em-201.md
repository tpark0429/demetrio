# Window Function
신호 처리 (Signal Processing) 분야에서 FFT (Fast Fourier Transform) 분석을 수행할 때, 시간 영역의 신호를 주파수 영역으로 변환하기 위해 사용되는 가중치 함수를 의미한다.

FFT는 전체 시간이 아닌 일정 프레임 단위의 신호만 보기 때문에 잘려 나가는 신호의 양 끝에서 불연속 (Discontiuous)이 발생하게 되며 이 불연속성은 주파수 스펨트럼에서 노이즈처럼 동작하게 된다. 이러한 현상을 **Spectral leakage**라고 부른다.

![spectral leakage](/assets/spectral_leakage.png)

따라서 이러한 문제를 해결하기 위해 왜곡의 원인이 되는 waveform의 전반부와 후반부의 가중치를 줄여 전체적인 spectrum의 왜곡을 완화시키는 수학적 기법을 **Window Function**이라고 부른다.

![window function](/assets/window_function.png)

단, RBW (Resolution Band Width) 개념과는 다르다. EMI 관점에서 보면 **RBW**는 정확한 주파수 포인트에서의 샘플링을 원하기 때문에 (예를들어 100 MHz 대역에 대한 측정을 원한다면 99.94 ~ 100.06 MHz 정도의 신호만 통과시켜 전력 측정) RBW 필터가 필요하며, 이에 비해 Window Function은 특정 포인트 샘플링이 아닌 일정 시간 단위에서의 FFT 기반 측정에서 필연적으로 발생하는 왜곡 성분을 줄이겠다는 의도이다.
