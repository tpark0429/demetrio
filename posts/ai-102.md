## Stochastic

### In Machine Learning

- Output이 randomness나 uncertainty를 가진 경우를 다룬다.
- Machine Learning 알고리즘은 명시적으로 randomness를 최적화 과정에 사용하기 때문에 output 또한 randomness나 uncertainty를 가질 수밖에 없다.
- 모든 경우의 수를 수식으로 완전히 전개하기 어려운 문제에서는 stochastic approach가 매우 강력하다.

### Stochastic Approach

- 확률적 접근법이라 써도 무방하다.
- Output이 나오기까지 어떤 randomness를 담고 있다고 보는 접근이다.
- Output이 사전에 완전히 결정되어 있는 값이라기보다 확률적 요소에 의해 생성된다고 보는 접근 방식이다.
- 복잡한 수식을 직접 전개하는 대신, random sampling을 반복하여 관심 있는 확률이나 기대값을 근사한다.

> 💡 **Mathematical Insight**: Stochastic approach의 핵심은 복잡한 확률 문제를 무조건 닫힌 형태의 수식으로 풀려고 하지 않고, 반복 sampling으로 확률 분포의 성질을 근사하는 것이다. Deep Learning에서 dataset을 통해 loss의 기대값을 근사하고 parameter를 업데이트하는 논리도 이와 연결된다.

## Birthday Problem 예제

### 문제 정의

100명이 한 공간에 있다고 가정한다. 윤년은 고려하지 않고, 생일은 365일 중 하나에 균등하게 분포한다고 둔다.

관심 있는 사건은 다음과 같다.

- 적어도 2명이 같은 생일을 가질 확률
- 적어도 3명이 같은 생일을 가질 확률
- 적어도 4명이 같은 생일을 가질 확률

2명이 같은 생일을 가질 확률은 여집합을 이용하면 비교적 쉽게 구할 수 있다. 즉, 모든 사람이 서로 다른 생일을 가질 확률을 먼저 구한 뒤 1에서 빼면 된다.

$$
P(\text{at least 2}) = 1 - \frac{365 \times 364 \times \cdots \times (365 - n + 1)}{365^n}
$$

100명일 때는 다음과 같다.

$$
P(\text{at least 2}) = 1 - \frac{365 \times 364 \times \cdots \times 266}{365^{100}}
$$

이 값은 거의 1에 가깝다. 따라서 100명이 있으면 같은 생일을 가진 사람이 적어도 2명 존재할 확률은 사실상 거의 확실하다.

### 3명 이상부터 어려워지는 이유

적어도 3명이 같은 생일을 가질 확률은 단순히 2명 문제처럼 쉽게 여집합을 계산하기 어렵다. 여집합은 "어떤 날짜에도 3명 이상이 몰리지 않는다"는 조건이다. 즉, 365개의 날짜 각각에 사람이 0명, 1명, 또는 2명까지만 배정되어야 한다.

적어도 4명이 같은 생일을 가질 확률의 여집합은 "어떤 날짜에도 4명 이상이 몰리지 않는다"는 조건이다. 이때 각 날짜에는 0명, 1명, 2명, 또는 3명까지만 배정될 수 있다.

이를 일반화하면 $n$명의 사람이 있고, 같은 생일을 가진 사람이 적어도 $k$명 존재할 확률은 다음과 같이 표현할 수 있다.

$$
P(M \ge k) = 1 - \frac{n!}{365^n} [x^n] \left( \sum_{j=0}^{k-1} \frac{x^j}{j!} \right)^{365}
$$

- $M$ : 같은 생일을 가진 사람 수의 최댓값
- $[x^n]$ : 다항식에서 $x^n$의 계수
- $k$ : 같은 생일을 공유하는 최소 인원 수

수식으로는 표현 가능하지만, $k=3$, $k=4$로 넘어갈수록 직접 손으로 계산하기는 매우 번거롭다. 이 지점에서 stochastic simulation이 실용적인 대안이 된다.

## Monte Carlo Simulation

### Python Example

아래 코드는 100명의 생일을 무작위로 생성하고, 같은 생일을 가진 사람이 적어도 $k$명 존재하는지를 반복 실험으로 추정한다.

```python
import random
import math

def same_birthday_prob(
    num_people: int,
    num_same: int
    ) -> bool:
    '''num_same 보다 많은 생일이 있는지 판별'''
    possible_date = range(365) # 2월 29일 포함
    birthdays = [0] * 365
    for _ in range(num_people):
        birth_date = random.choice(possible_date)
        birthdays[birth_date] += 1
    return max(birthdays) >= num_same # return boolean


def birthday_estimator(
    num_people: int,
    num_same: int,
    trials: int) -> float:
    '''num_people 중에 생일이 같은 사람이 있을 확률 계산'''
    hit_count = 0
    for _ in range(trials):
        if same_birthday_prob(num_people, num_same):
            hit_count += 1
    return hit_count/trials


def run_birthday_simulation(
    num_same: int,
    peoples: list) -> None:
    '''같은 생일이 있는지 시뮬레이션 수행'''
    for people in peoples:
        print('---'*10)
        estimated_probability = round(birthday_estimator(people, num_same, 1000), 3)
        numerator = math.factorial(365)
        denominator = (365**people)*math.factorial(365-people)
        theoretical_prob = round(1 - numerator/denominator, 3)
        print(f'{people}명 중 같은 생일이 있을 확률: \n\
시뮬레이션: {estimated_probability}\t이론값: {theoretical_prob}')



if __name__=='__main__':
    run_birthday_simulation(2, [10, 20, 40, 100])
```

### Simulation Result

200,000번 반복 실험한 결과는 다음과 같다.

| Event | Simulation | Theory |
|---|---:|---:|
| 적어도 2명이 같은 생일 | 1.000000 | 0.9999997 |
| 적어도 3명이 같은 생일 | 0.645775 | 0.6458645 |
| 적어도 4명이 같은 생일 | 0.064425 | 0.0635756 |
| 적어도 5명이 같은 생일 | 0.003415 | 0.0034116 |

실험 횟수가 충분히 크면 stochastic simulation 결과가 이론값에 매우 가깝게 수렴한다. 특히 3명, 4명, 5명처럼 수식을 직접 전개하기 까다로운 경우에도 코드는 거의 동일하게 유지된다. 달라지는 것은 `threshold` 값뿐이다.

## Deep Learning에서의 Stochastic

Deep Learning에서도 stochastic approach가 핵심적인 역할을 한다. 우리가 최소화하고 싶은 것은 전체 데이터 분포에 대한 loss의 기대값이다.

$$
\mathcal{L}(\theta)=\mathbb{E}_{(x,y)\sim P_{\text{data}}}\left[\ell(f_\theta(x),y)\right]
$$

실제 데이터 분포 $P_{\text{data}}$를 완벽히 알 수 없으므로, 크기가 $N$인 training dataset으로 기대 loss를 근사한다.

$$
\mathcal{L}_N(\theta)=\frac{1}{N}\sum_{i=1}^{N}\ell(f_\theta(x_i),y_i)
$$

gradient descent는 현재 parameter에서 loss가 가장 빠르게 증가하는 방향의 반대쪽으로 parameter를 이동한다.

$$
\theta_{t+1}=\theta_t-\eta g_t
$$

여기서 $\eta$는 learning rate이고, $g_t$는 $t$번째 step에서 계산한 gradient이다. 한 번의 update에 몇 개의 sample을 사용하는지에 따라 full batch, SGD, mini-batch 방식으로 구분한다.

![Full batch, SGD, mini-batch의 gradient 계산 비교](/assets/deep_learning_batch_comparison.svg)

### Full Batch Gradient Descent

Full batch gradient descent는 한 번의 update마다 dataset의 $N$개 sample을 모두 사용한다.

$$
g_t=\frac{1}{N}\sum_{i=1}^{N}\nabla_\theta\ell(f_{\theta_t}(x_i),y_i)
$$

전체 dataset에 대한 정확한 gradient를 사용하므로 같은 parameter와 dataset에서는 update 방향이 항상 같다. gradient가 안정적이라는 장점이 있지만, dataset이 커질수록 한 번의 update에 필요한 계산량과 memory 사용량이 커진다. 또한 모든 sample의 계산이 끝나기 전까지 parameter를 갱신할 수 없다.

### Stochastic Gradient Descent

엄밀한 의미의 stochastic gradient descent, 즉 SGD는 매 update마다 dataset에서 sample 하나 $(x_{i_t},y_{i_t})$를 무작위로 선택하여 gradient를 계산한다.

$$
g_t=\nabla_\theta\ell(f_{\theta_t}(x_{i_t}),y_{i_t})
$$

선택되는 sample $i_t$가 매번 달라지므로 $g_t$도 확률변수가 된다. 개별 gradient는 전체 dataset의 gradient와 다를 수 있지만, sample을 균등하게 추출한다면 그 기대값은 full batch gradient와 일치한다.

$$
\mathbb{E}_{i_t}[g_t]
=\frac{1}{N}\sum_{i=1}^{N}\nabla_\theta\ell(f_{\theta_t}(x_i),y_i)
=\nabla_\theta\mathcal{L}_N(\theta_t)
$$

따라서 SGD는 전체 gradient를 정확히 계산하는 대신 계산 비용이 작은 noisy gradient를 반복적으로 사용한다. update가 빠르고 gradient noise가 saddle point나 평평한 구간을 벗어나는 데 도움을 줄 수 있지만, variance가 커서 loss와 parameter의 이동 경로가 크게 흔들릴 수 있다.

### Mini-batch Gradient Descent

실제 Deep Learning에서는 sample 하나만 사용하는 순수 SGD보다, $B$개의 sample로 구성된 mini-batch $\mathcal{B}_t$를 사용하는 경우가 일반적이다.

$$
g_t=\frac{1}{B}\sum_{i\in\mathcal{B}_t}
\nabla_\theta\ell(f_{\theta_t}(x_i),y_i)
$4

mini-batch는 full batch와 SGD의 절충안이다. 여러 sample의 gradient를 평균하므로 sample 하나를 사용할 때보다 variance가 작고, 전체 dataset을 사용할 때보다 빠르게 update할 수 있다. 행렬 연산과 GPU 병렬 처리도 효율적으로 이용할 수 있다.

| 방식 | 한 번의 update에 사용하는 sample | Gradient 특성 | 주요 특징 |
|---|---:|---|---|
| Full batch | $N$개 | 정확하고 안정적 | update당 계산량과 memory 사용량이 크다. |
| SGD | 1개 | noise와 variance가 크다. | update가 매우 빠르지만 학습 경로가 크게 흔들린다. |
| Mini-batch | $B$개, $1<B<N$ | 적당한 noise와 안정성 | 계산 효율과 gradient 품질의 균형이 좋다. |

Deep Learning 문헌과 framework에서는 mini-batch gradient descent까지 관습적으로 SGD라고 부르는 경우가 많다. 예를 들어 optimizer를 `SGD`로 설정하고 batch size를 32로 두었다면, 실제로는 32개 sample의 평균 gradient로 parameter를 update한다.

### Epoch, Batch, Iteration

- **Epoch**: 전체 training dataset을 한 번 모두 사용하는 단위이다.
- **Batch 또는 mini-batch**: 한 번의 update에 사용하는 sample의 묶음이다.
- **Iteration 또는 step**: 한 batch로 gradient를 계산하고 parameter를 한 번 update하는 과정이다.

dataset 크기가 $N$이고 batch size가 $B$라면, 한 epoch에는 대략 $\lceil N/B\rceil$번의 iteration이 발생한다. 보통 각 epoch가 시작될 때 dataset을 shuffle하여 매번 서로 다른 mini-batch 조합을 만든다. 이 sampling과 shuffle이 학습 과정에 stochasticity를 만든다.

### Batch Size와 Stochasticity

- 작은 batch는 update가 자주 일어나고 gradient noise가 크다. memory 사용량은 작지만 병렬 연산의 효율이 낮을 수 있다.
- 큰 batch는 gradient가 full batch gradient에 가까워져 안정적이다. 다만 더 많은 memory가 필요하고, batch size를 키울 때 learning rate 조정이 필요할 수 있다.
- 적절한 mini-batch는 hardware 효율, 학습 안정성, generalization을 함께 고려하여 선택한다.

생일 문제에서 손으로 수식을 전개하는 대신 Monte Carlo simulation으로 확률을 근사한 것처럼, SGD도 전체 데이터에 대한 계산을 작은 random sample의 반복 계산으로 대체한다. 각 update는 불완전하고 흔들리지만, 반복하면 전체 loss를 줄이는 방향으로 학습이 진행된다.

## Stochastic Process
- 시간의 흐름에 따라 발생하는 확률적 변화 과정
- Random 실험 결과를 시간에 따른 값으로 mapping 해주는 함수

### Ensemble
- Random process로부터 나타나는 시행 결과들의 모음 (집합)


확률과정(stochastic process)은 시간 집합 $T$와 표본공간 $\Omega$에 대하여 다음과 같이 나타낼 수 있다.

$$
X:T\times\Omega\rightarrow\mathbb{R},\qquad (t,\zeta)\mapsto X(t,\zeta)
$$

여기서 $t\in T$는 시간, $\zeta\in\Omega$는 한 번의 실험 결과를 나타낸다. 문헌에 따라 $X(\zeta,t)$로 쓰기도 하며, 이 절에서는 두 표기가 같은 대상을 뜻한다고 본다.

어떤 실험을 독립적으로 $n$번 시행했다고 하자. $i$번째 시행의 시간에 따른 결과는 $x_i(t)=X(t,\zeta_i)$이고, 다음 함수들의 모음을 **ensemble**이라고 한다.

$$
\mathcal{E}_n=\left\{X(t,\zeta_1),X(t,\zeta_2),\ldots,X(t,\zeta_n)\right\}
$$

![확률과정의 ensemble, sample function, random variable](/assets/stochastic_process_ensemble.svg)

#### $\zeta$를 고정하면: deterministic function

실험 결과 하나를 $\zeta=\zeta_i$로 고정하면 $X_{\zeta_i}(t)=X(t,\zeta_i)$가 된다. 한 번의 시행 결과가 정해졌으므로 각 시간의 값도 하나의 곡선으로 정해진다. 따라서 이는 $t$에 대한 **deterministic function**이며, sample function, sample path 또는 realization이라고 부른다. 그림의 파란색 곡선 하나가 이에 해당한다.

#### $t$를 고정하면: random variable

반대로 특정 시각 $t=t_0$를 고정하면 $X_{t_0}(\zeta)=X(t_0,\zeta)$가 된다. 시행 결과 $\zeta$에 따라 값이 달라지므로 이는 표본공간 $\Omega$에서 실수로 가는 **random variable**이다. 그림의 붉은 점선과 만나는 점들이 다음 확률변수의 표본이다.

$$
\left\{X(t_0,\zeta_1),X(t_0,\zeta_2),\ldots,X(t_0,\zeta_n)\right\}
$$

ensemble 기법은 같은 시각 $t_0$에서 여러 시행의 값을 모아 통계량을 추정한다. 예를 들어 ensemble mean은

$$
\widehat{m}_X(t_0)=\frac{1}{n}\sum_{i=1}^{n}X(t_0,\zeta_i)
$$

이며, $n$이 충분히 크면 이론적인 평균 $m_X(t_0)=\mathbb{E}[X(t_0)]$을 근사한다. 즉, 같은 확률과정을 **가로로 보면 한 시행의 시간 변화**, **세로로 보면 한 시각에서의 확률분포**가 된다.

#### Example: Stochastic process on an exponential function

초기값 $X_0>0$은 고정되어 있지만, 지수함수의 성장률 $R$이 실험마다 무작위로 결정되는 상황을 생각해 보자.

$$
X(t,\zeta)=X_0\exp\left(R(\zeta)t\right),
\qquad R\sim\mathcal{N}(\mu,\sigma^2)
$$

예를 들어 $X(t,\zeta)$를 서로 다른 환경에서 배양한 미생물의 개체 수라고 보면, $R(\zeta)$는 온도나 영양 상태의 차이로 인해 매 시행마다 달라지는 성장률이다.

실험을 $n$번 시행하여 성장률 $r_i=R(\zeta_i)$를 얻었다면 ensemble은 다음과 같다.

$$
\mathcal{E}_n
=\left\{X_0e^{r_1t},X_0e^{r_2t},\ldots,X_0e^{r_nt}\right\}
$$

한 시행 결과 $\zeta_i$를 고정하면 $r_i$도 하나의 값으로 결정되므로

$$
X(t,\zeta_i)=X_0e^{r_it}
$$

는 시간 $t$에 대한 deterministic exponential function이다. $r_i>0$이면 증가하고, $r_i<0$이면 감소하지만, 어느 경우든 하나의 시행 안에서는 곡선이 완전히 정해져 있다.

반대로 시각 $t=t_0$를 고정하면

$$
X(t_0,\zeta)=X_0e^{R(\zeta)t_0}
$$

가 되어 시행마다 다른 값을 갖는 random variable이 된다. 특히 $R$이 정규분포를 따르면

$$
\log\frac{X(t_0)}{X_0}=R t_0
\sim\mathcal{N}\left(\mu t_0,\sigma^2t_0^2\right)
$$

이므로 $X(t_0)$는 log-normal distribution을 따른다. 이때 이론적인 ensemble mean은 정규분포의 moment-generating function을 이용하여

$$
m_X(t)=\mathbb{E}[X(t)]
=X_0\mathbb{E}\left[e^{Rt}\right]
=X_0\exp\left(\mu t+\frac{1}{2}\sigma^2t^2\right)
$$

로 구할 수 있다. 실제 $n$번의 시행에서는

$$
\widehat{m}_X(t)
=\frac{1}{n}\sum_{i=1}^{n}X_0e^{r_it}
$$

로 이를 추정한다. 따라서 각 실험에서 관측되는 것은 하나의 deterministic exponential curve이지만, 여러 실험을 모으면 성장률의 불확실성이 반영된 stochastic process의 ensemble이 된다.
