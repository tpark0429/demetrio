## Stochastic

### In Machine Learning

- Output이 randomness나 uncertainty를 가진 경우를 다룬다.
- Machine Learning 알고리즘은 명시적으로 randomness를 최적화 과정에 사용하기 때문에 output 또한 randomness나 uncertainty를 가질 수밖에 없다.
- 모든 경우의 수를 수식으로 완전히 전개하기 어려운 문제에서는 stochastic approach가 매우 강력하다.

### Stochastic Approach

- 확률적 접근법이라 써도 무방하다.
- Output이 나오기까지 어떤 randomness를 담고 있다고 보는 접근이다.
- Output이 사전에 완전히 결정되어 있는 값이라기보다 확률적 요소에 의해 생성된다고 보는 접근 방식이다.
- 복잡한 수식을 직접 전개하는 대신, random sampling을 반복하여 관심 있는 확률이나 기대값을 근사한다.

> 💡 **Mathematical Insight**: Stochastic approach의 핵심은 복잡한 확률 문제를 무조건 닫힌 형태의 수식으로 풀려고 하지 않고, 반복 sampling으로 확률 분포의 성질을 근사하는 것이다. Deep Learning에서 dataset을 통해 loss의 기대값을 근사하고 parameter를 업데이트하는 논리도 이와 연결된다.

## Birthday Problem 예제

### 문제 정의

100명이 한 공간에 있다고 가정한다. 윤년은 고려하지 않고, 생일은 365일 중 하나에 균등하게 분포한다고 둔다.

관심 있는 사건은 다음과 같다.

- 적어도 2명이 같은 생일을 가질 확률
- 적어도 3명이 같은 생일을 가질 확률
- 적어도 4명이 같은 생일을 가질 확률

2명이 같은 생일을 가질 확률은 여집합을 이용하면 비교적 쉽게 구할 수 있다. 즉, 모든 사람이 서로 다른 생일을 가질 확률을 먼저 구한 뒤 1에서 빼면 된다.

$$
P(\text{at least 2}) = 1 - \frac{365 \times 364 \times \cdots \times (365 - n + 1)}{365^n}
$$

100명일 때는 다음과 같다.

$$
P(\text{at least 2}) = 1 - \frac{365 \times 364 \times \cdots \times 266}{365^{100}}
$$

이 값은 거의 1에 가깝다. 따라서 100명이 있으면 같은 생일을 가진 사람이 적어도 2명 존재할 확률은 사실상 거의 확실하다.

### 3명 이상부터 어려워지는 이유

적어도 3명이 같은 생일을 가질 확률은 단순히 2명 문제처럼 쉽게 여집합을 계산하기 어렵다. 여집합은 "어떤 날짜에도 3명 이상이 몰리지 않는다"는 조건이다. 즉, 365개의 날짜 각각에 사람이 0명, 1명, 또는 2명까지만 배정되어야 한다.

적어도 4명이 같은 생일을 가질 확률의 여집합은 "어떤 날짜에도 4명 이상이 몰리지 않는다"는 조건이다. 이때 각 날짜에는 0명, 1명, 2명, 또는 3명까지만 배정될 수 있다.

이를 일반화하면 $n$명의 사람이 있고, 같은 생일을 가진 사람이 적어도 $k$명 존재할 확률은 다음과 같이 표현할 수 있다.

$$
P(M \ge k) = 1 - \frac{n!}{365^n} [x^n] \left( \sum_{j=0}^{k-1} \frac{x^j}{j!} \right)^{365}
$$

- $M$ : 같은 생일을 가진 사람 수의 최댓값
- $[x^n]$ : 다항식에서 $x^n$의 계수
- $k$ : 같은 생일을 공유하는 최소 인원 수

수식으로는 표현 가능하지만, $k=3$, $k=4$로 넘어갈수록 직접 손으로 계산하기는 매우 번거롭다. 이 지점에서 stochastic simulation이 실용적인 대안이 된다.

## Monte Carlo Simulation

### Python Example

아래 코드는 100명의 생일을 무작위로 생성하고, 같은 생일을 가진 사람이 적어도 $k$명 존재하는지를 반복 실험으로 추정한다.

```python
import random
from collections import Counter


def estimate_birthday_probability(
    n_people: int = 100,
    threshold: int = 2,
    trials: int = 200_000,
    days: int = 365,
    seed: int = 42,
) -> float:
    rng = random.Random(seed)
    hit = 0

    for _ in range(trials):
        birthdays = [rng.randrange(days) for _ in range(n_people)]
        max_same_birthday = max(Counter(birthdays).values())

        if max_same_birthday >= threshold:
            hit += 1

    return hit / trials


for k in [2, 3, 4, 5]:
    probability = estimate_birthday_probability(threshold=k)
    print(f"at least {k}: {probability:.6f}")
```

### Simulation Result

200,000번 반복 실험한 결과는 다음과 같다.

| Event | Simulation | Theory |
|---|---:|---:|
| 적어도 2명이 같은 생일 | 1.000000 | 0.9999997 |
| 적어도 3명이 같은 생일 | 0.645775 | 0.6458645 |
| 적어도 4명이 같은 생일 | 0.064425 | 0.0635756 |
| 적어도 5명이 같은 생일 | 0.003415 | 0.0034116 |

실험 횟수가 충분히 크면 stochastic simulation 결과가 이론값에 매우 가깝게 수렴한다. 특히 3명, 4명, 5명처럼 수식을 직접 전개하기 까다로운 경우에도 코드는 거의 동일하게 유지된다. 달라지는 것은 `threshold` 값뿐이다.

## Deep Learning과의 연결

Deep Learning에서도 비슷한 일이 일어난다. 우리가 정말 알고 싶은 것은 전체 데이터 분포에 대한 loss의 기대값이다.

$$
\mathcal{L}(\theta) = \mathbb{E}_{(x,y) \sim P_{\text{data}}} [\ell(f_\theta(x), y)]
$$

그러나 실제 데이터 분포 $P_{\text{data}}$를 완벽히 알 수 없고, 가능한 모든 입력과 정답 조합을 수식으로 전개할 수도 없다. 따라서 dataset에서 mini-batch를 sampling하여 loss의 기대값을 근사한다.

$$
\hat{\mathcal{L}}(\theta) = \frac{1}{B} \sum_{i=1}^{B} \ell(f_\theta(x_i), y_i)
$$

- $B$ : mini-batch size
- $(x_i, y_i)$ : dataset에서 sampling된 입력과 정답
- $\theta$ : neural network의 parameter

즉, Deep Learning은 "복잡한 현실의 분포를 완전한 수식으로 먼저 푼 뒤 적용하는 방식"이 아니다. 데이터에서 반복적으로 sampling하고, 그 sampling 결과를 이용해 loss와 gradient를 근사하며, parameter를 업데이트하는 방식이다.

생일 문제에서 손으로 수식을 전개하는 대신 Monte Carlo simulation으로 확률을 근사할 수 있었던 것처럼, Deep Learning도 현실의 복잡한 데이터 분포를 stochastic하게 학습한다. 이것이 데이터 기반 학습이 수식 기반 풀이보다 실용적인 경우가 많은 이유이다.
